# Architecture

The application runs entirely on a single Nirvana virtual machine using Docker Compose. Five services share the host:

- **app**: The FastAPI process serving the `/chat`, `/ingest`, and `/health` endpoints. Built from the `app/` directory in this repository.
- **qdrant**: The vector database that holds embedded document chunks. Vectors are 384-dimensional, generated by the FastEmbed model `BAAI/bge-small-en-v1.5`.
- **postgres**: Persists LangGraph conversation checkpoints. Each session is keyed by its `session_id` and identified internally as a LangGraph `thread_id`.
- **redis**: Tracks lightweight per-session activity counters (turn count, last-active timestamp).
- **ollama**: Serves the local large language model. The default `qwen2.5:3b-instruct` is pulled on first start and cached in a Docker volume. No external LLM API is used.

The four services communicate over an internal Docker bridge network. Only port 8000 (the app) is exposed to the public internet; everything else is reachable only from inside the host.

The application uses `langgraph` to coordinate retrieval and generation as an explicit state machine with two nodes: a `retrieve` node that queries Qdrant, and a `generate` node that calls the Anthropic API. This makes it straightforward to extend the agent with additional steps such as query reformulation, tool use, or human-in-the-loop review.
